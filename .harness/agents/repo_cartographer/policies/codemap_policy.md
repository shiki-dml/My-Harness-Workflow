# CODEMAP Policy

## Purpose of CODEMAP.md
`CODEMAP.md` describes repository structure, file purposes, modification boundaries, restricted paths, unknown areas, and notes for downstream agents.

## Create When Missing
If `CODEMAP.md` does not exist, `repo_cartographer` may create it using the approved template and managed section markers.

## Managed Section Markers
Managed content must appear only between:

```html
<!-- BEGIN REPO_CARTOGRAPHER_MANAGED -->
<!-- END REPO_CARTOGRAPHER_MANAGED -->
```

## Existing CODEMAP Handling
If `CODEMAP.md` already exists and contains managed section markers, refresh only the managed sections. If it does not contain managed section markers, treat it as human-authored and block before overwriting.

## Human-Authored Content Preservation
Human-authored content outside managed sections must be preserved. The agent must never remove human notes outside the managed section.

## Conflict Behavior
If content conflict is detected, block the update and request human approval through `harness_orchestrator`. Missing approval is not approval.

## Required CODEMAP Sections
`CODEMAP.md` should include:

- Purpose.
- Generated By.
- Last Updated.
- Repository Summary.
- Top-Level Structure.
- Harness Agents.
- Project Governance Files.
- Documentation Files.
- Scripts and Tooling.
- Source Code Areas.
- Test Areas.
- Configuration Files.
- Generated, External, and Ignored Paths.
- Restricted or Sensitive Paths.
- Unknown or Unmapped Areas.
- File Ownership and Modification Rules.
- Notes for Downstream Agents.
- Human Notes.

## Validation Requirements
Validate that `CODEMAP.md` contains the managed markers, preserves human-authored content outside managed sections, does not expose secrets, and does not include product plans, backlog items, sprint contracts, implementation work, or QA results.
